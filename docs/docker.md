# 🐳 Docker

This project uses Docker to provide a consistent development environment.

## Why Docker?

- No local installation of MyST
- Same environment for every developer
- Easy onboarding
- Reproducible builds

## Start the project

```bash
make up
```

## Stop the project

```bash
make down
```

## Open a shell

```bash
make shell
```

## View logs

```bash
make logs
```
